# GymTrack — iPhone 主屏小组件 + GZCLP 课表 (Scriptable)

跑 **GZCLP**（4 天循环 A1/B1/A2/B2），每周三次按固定星期（默认**一三五**）滚动。
主屏小组件随时看「今天/明天/后天 该不该练、练哪些动作」；点开弹菜单**完成 / 顺延一天 / 设置**。
单位 **lb**。静态显示动作（组×次），**重量自己记**，App 不自动进阶。数据存 iCloud，跨设备同步。

## 课表（家庭杠铃+哑铃版）
`+` = 末组 AMRAP（尽力做）。同一动作的 **T1 与 T2 是两套独立重量**，分开记。

| Day | T1 主项 5×3+ | T2 副项 3×10 | T3 辅助 3×15+ |
|-----|--------------|--------------|----------------|
| A1 | 深蹲 Squat | 卧推 Bench | 引体 Pull-up |
| B1 | 推举 OHP | 硬拉 Deadlift | 引体 Pull-up |
| A2 | 卧推 Bench | 深蹲 Squat | 引体 Pull-up |
| B2 | 硬拉 Deadlift | 推举 OHP | 引体 Pull-up |

### 进阶规则（自己照着加重，App 只显示动作不自动算）
- **T1**（5×3+）：每次成功就加重——下肢（深蹲/硬拉）+10 lb，上肢（卧推/推举）+5 lb。
  没完成所有次数 → 降阶 `5×3 → 6×2 → 10×1`；`10×1` 再失败 → 用新 5RM 重置回 `5×3`。
- **T2**（3×10）：完成全部次数就加重（同上幅度）。失败 → `3×10 → 3×8 → 3×6`；`3×6` 再失败 → 减重重置回 `3×10`。
- **T3**（3×15+）：末组 ≥25 次 → 下次加重。
- 卡重时改用微调片（1.25 lb/片，杠两端共 +2.5 lb）。

> 你的器械（深蹲架/卧推凳/引体杆/可调哑铃）能完整覆盖以上动作，无需替换。

## 文件
- `GymTrack.js` —— 全部逻辑（单文件，放进 Scriptable）。
- `test/schedule.test.js` —— 调度/顺延逻辑的 Node 自检（`node test/schedule.test.js`，31 断言）。

## 安装到 iPhone
1. App Store 装 **Scriptable**（免费）。
2. 把 `GymTrack.js` 放进 `iCloud Drive/Scriptable/`，或在 Scriptable 里 + 新建脚本整段粘贴，命名 **GymTrack**。
3. 打开 Scriptable 点 ▶︎ Run：弹菜单、并生成 `gymtrack.json`。
4. 回主屏，长按空白 → 加 **Scriptable** 小组件（建议 **medium**，能列出当天三个动作）。
5. 长按该小组件 → **Edit Widget**：Script=`GymTrack`，When Interacting=**Run Script**。

## 日常用法
- **看**：主屏小组件显示今天该练哪天 + T1/T2/T3 三个动作（medium/large）。
- **练完**：点小组件 → 「✅ 完成今天」→ 自动排下一次（回到一三五网格）。
- **今天练不了**：点小组件 → 「⏭️ 顺延一天」。
  - 仍在空档日（如周一→周二）只动这一次，周三/周五不变。
  - 连续顺延撞上下一个固定槽时吸附回网格、取代那一次，其余顺次后移，**尽量保持一三五**。
- **改训练星期**：点小组件 → 「⚙️ 设置」→ 改训练星期。
  **改课表/动作**：编辑脚本顶部的 `DEFAULTS.program`，**改完即生效**（课表由代码定义，不存在 json 里）。

## 数据分两处
- **课表（`program`、`units`）**：写在脚本顶部 `DEFAULTS`，是「配置」。改它立即生效。
- **进度**：存 `iCloud Drive/Scriptable/gymtrack.json`，只含可变状态：
```jsonc
{
  "weekdays": [1, 3, 5],   // 0=周日 1=周一 … 6=周六（设置里可改）
  "programIndex": 0,        // 下一次练第几个 (0 起)
  "nextDate": "2026-06-29",
  "history": [ { "date": "2026-06-29", "dayName": "A1 (推举日)" } ]
}
```
> 所以改课表只动脚本即可，不用删 json，进度也不会丢。
